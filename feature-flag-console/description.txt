3. Evaluation SDK / Middleware
   - Lightweight helper that provides deterministic evaluations using a hash-based rollout mechanism.
   - Ensures resilience with graceful fallbacks to MongoDB when Redis is unavailable.

âš™ï¸ Key Features
- Flag Models: Boolean & multivariate flags with metadata (name, owner, tags, description, lifecycle states).
- Targeting Rules: Environment scoping (dev/stage/prod), user attributes (role, plan, region), and percentage rollouts using deterministic hashing.

- Admin UI: Create/edit flags, define rollout rules, simulate evaluations, trigger kill switches with confirmation.
- Audit & Governance: Immutable logs for all flag changes with who/what/when details, diff view for rule changes, and RBAC for access control.
- Observability: Metrics for evaluation rate, cache hits/misses, error tracking, and exportable JSON configuration.
- Security: JWT-protected APIs, input sanitization, scoped secrets, and ETag-based caching for clients.
- APIs: REST endpoints for fetching, evaluating, and mutating flags, including batch evaluation and streaming updates.
- Resilience & Testing: Rate limiting, MongoDB fallback on cache miss, integration tests, and Docker Compose setup (Redis + MongoDB + API + UI).
